# Logs Directory

This directory contains log files generated by the web scraping projects.

## Log Files

Each scraper module creates its own rotating log file:
- `ecommerce_tracker.log` - E-commerce price tracker logs
- `job_scraper.log` - Job portal scraper logs
- `news_aggregator.log` - News headline aggregator logs
- `realestate_crawler.log` - Real estate crawler logs

## Log Rotation

Logs use rotating file handlers with the following settings:
- Maximum size per file: 10MB
- Backup files kept: 5
- Total maximum storage: ~50MB per scraper

Old log files are automatically archived with numeric suffixes (e.g., `ecommerce_tracker.log.1`, `ecommerce_tracker.log.2`, etc.)

## Log Format

Each log entry includes:
- Timestamp (YYYY-MM-DD HH:MM:SS)
- Logger name
- Log level (DEBUG, INFO, WARNING, ERROR, CRITICAL)
- Message

Example:
```
2026-01-20 16:20:00 - EcommerceTracker - INFO - Starting Amazon scrape
```

## Cleanup

You can safely delete old log files when troubleshooting is complete. This directory is git-ignored.
